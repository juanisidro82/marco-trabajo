# PyCharm: Workflow Practices

PyCharm operates as the **Workshop**: the place where what the agents produced is reviewed and refined, and where surgical edits that require precision, not broad context, are made.

---

## 1. Review Agent Output Before Commit

Before performing `git commit`, open the diff in PyCharm:
- `Git` → `Show Diff` (or `Cmd/Ctrl + D`) on the modified file.
- Read every change line by line — agents often introduce:
  - Unnecessary imports.
  - Inconsistently renamed variables.
  - Duplicated logic that already existed in another method.
  - Incorrect or missing type hints.

**Rule:** If the agent touched more than 3 files, review them all before committing.

---

## 2. Surgical Refactoring with Native Tools

When an agent's method is good but poorly named, or a class has grown too large:

### Safe Rename
- Right-click on the name → `Refactor` → `Rename` (`Shift + F6`).
- PyCharm automatically updates **all** references.
- Never rename with manual find/replace — it breaks imports and strings.

### Extract Method
- Select a block of code within a long method.
- `Refactor` → `Extract` → `Method` (`Cmd/Ctrl + Alt + M`).
- Useful when processing functions grow too large.

### Move Class to Another File
- Right-click on the class → `Refactor` → `Move`.
- PyCharm updates imports automatically.

---

## 3. Type Inspection with Type Hints

Agents generally omit type hints. Adding them in PyCharm activates static validation:

```python
# Without type hints (typical agent output)
def get_client(self, client_id, include_payments=None):

# With type hints (reviewed in PyCharm)
def get_client(self, client_id: int, include_payments: bool = False) -> dict | None:
```

- `Code` → `Inspect Code` detects type inconsistencies throughout the file.
- Red-underlined type errors are potential bugs — do not ignore them.

---

## 4. Debugger to Verify API/DB Responses

When an agent produces code that calls an API or DB but the result is unexpected:
1. Place a breakpoint on the `return` line within the query function.
2. Run in debug mode (`Shift + F9`).
3. Inspect the result directly in the variables panel.

This is faster than adding `print()` and more informative than reading JSON in the terminal.

---

## 5. Local History as a Safety Net

PyCharm saves a local history of every file, independent of Git:
- Right-click on the file → `Local History` → `Show History`.
- Allows recovery of previous versions even if you haven't committed.

Useful when an agent overwrote something that worked and you've already closed the session.

---

## 6. Find Usages Before Changing a Method

Before modifying or deleting any method:
- Right-click → `Find Usages` (`Alt + F7`).
- Shows all places where that method is called.

Prevents breaking other modules when refactoring.

---

## 7. JetBrains AI — How to Use it Correctly

JetBrains AI is useful as an **inline suggestion**, not as an autonomous agent.

### Do use for:
- Completing a method you've already started writing.
- Explaining a block of code generated by an agent that you don't understand.
- Suggesting the correct type hint for a parameter.

### Do NOT use for:
- Asking it to design a complete module.
- Asking it to modify logic that touches multiple files.
- Replacing Claude Code for architectural decisions.

**Golden Rule:** If the JetBrains AI response requires more than 20 lines, the task is too large for the Workshop — pass it to the Specialists.
